# Contributing

Contributions are welcome when they make workflows more practical and easier to maintain.

## Good Contributions

- New workflow examples
- Clearer task-card fields
- Better verification checklists
- Tool-routing improvements
- Editable asset patterns

## Style

- Keep language direct.
- Prefer examples over theory.
- Include source files when possible.
- Avoid adding unnecessary tools.

## Pull Request Checklist

- [ ] The workflow has a clear final artifact.
- [ ] Editable source files are named.
- [ ] Verification steps are included.
- [ ] Manual boundaries are documented.
- [ ] The change is small enough to review.
