# Codex Plugin Stack For Solo Builders

The best plugin stack is not the biggest one.

For a solo builder, the goal is to reduce decision friction.

## Core Stack

| Plugin | Use |
| --- | --- |
| Browser | Verify local web pages, dashboards, and rendered HTML |
| Documents | Write reports, SOPs, and deliverable documents |
| Presentations | Create decks, product explainers, and course materials |
| Spreadsheets | Analyze products, competitors, finances, and growth data |
| Hyperframes | Create videos, motion assets, and website-to-video workflows |

## Why This Stack Works

It covers the common output types:

- Web UI
- Documents
- Slides
- Tables
- Video

It also avoids overlapping tools.

## Plugins To Delay

Delay these until the task clearly needs them:

- Design collaboration tools
- Scientific illustration tools
- Marketing-data connectors
- Duplicate video frameworks
- Social-media design tools already handled outside Codex

## Plugin Selection Rule

Install a plugin only when at least one of these is true:

- The task repeats every week.
- The plugin improves output quality.
- The plugin makes verification easier.
- The plugin removes a real manual bottleneck.

Do not install a plugin just because a post listed it.

## Example Decision

If you already use an HTML/CSS rendering workflow for content cards, you may not need a design plugin inside Codex.

Keep the main chain simple:

```text
Markdown or JSON -> HTML/CSS template -> Browser verification -> exported asset
```

The source file matters more than the shiny tool.
