# Wiki Rules

## Source Handling

- Read from `raw/`.
- Do not modify `raw/` files.
- Write clean Markdown into `wiki/`.
- Preserve important source details.
- Add links between related notes.

## Note Format

Each wiki note should include:

- Title
- Summary
- Key points
- Related links
- Open questions if needed

## Link Rules

Use Obsidian-style internal links:

```text
[[Codex_Setup]]
[[Plugin_Audit]]
[[Editable_Assets]]
```

## Conflict Rules

If a new source conflicts with an existing wiki note:

1. Do not overwrite silently.
2. Add a conflict note.
3. Ask for human review if the difference matters.

## Update Rule

Prefer small updates.

Do not rewrite the whole wiki unless the structure is wrong.
